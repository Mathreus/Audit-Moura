<h1>Auditoria Interna Moura: Repositório de Testes Automáticos</h1> 

# Bem vindos, Auditores!

Este repositório é a central de automação da Auditoria Interna. Nosso objetivo é transformar a forma como trabalhamos, substituindo tarefas manuais repetitivas por análises rápidas, consistentes e escaláveis, permitindo que nossa equipe dedique mais tempo a atividades de alto valor e tenhamos um salto de produtividade! 

Aqui, você encontrará os códigos e scripts desenvolvidos para auditar nossos distribuidores e processos de forma inteligente. Vamos construir, juntos, uma auditoria mais produtiva, ágil e precisa.

# Estrutura e Organização do Repositório:

# 1) Organização por Áreas:

Os testes estão organizados por áreas e contas contábeis específicas, com o objetivo de agilizar a localização do script adequado para cada análise. As áreas disponíveis são:

  - Adiantamento a Fornecedor;
  - BIN
  - Caixa
  - Cartão Presente
  - Certidões
  - Contas a Receber
  - Estoques
  - Folha de Pagamento
  - Fornecedores
  - Governança Corporativa
  - Imobilizado
  - Receita
    
# 2) Sistema de Codificação dos Testes:
Cada teste automatizado possui um código único e descritivo que segue o padrão [SIGLA_ÁREA].[NÚMERO], exemplo:

Um teste de estoque qualquer pode ser denomidado como "EST.01"

Essa organização  melhora nossa:
- Rastreabilidade: Facilita referenciar o teste em papéis de trabalho e relatórios.
- Organização: Auxilia no controle de quais rotinas já foram automatizadas.
- Clareza: A sigla já indica a qual área o teste pertence.

# 3) Para executar um teste:

- Navegue até a pasta da área de interesse;
- Localize o script com o código do teste desejado;
- Copie o código do teste e cole no ambiente de programação utilizado para executar o teste
- Leia e siga as instruções escritas no início do código, lá existem dicas de como executar de maneira correta
- É importante destacar que os códigos podem necessitar de atualizações e refinamentos contínuos para se manterem alinhados com eventuais mudanças nos processos auditados e assegurar a precisão e confiabilidade dos resultados!

# 4) Resultados esperados:

A implementação bem-sucedida desta biblioteca de automação busca:

- Reduzir o tempo gasto em testes manuais, demorados e repetitivos;
- Aumentar a cobertura de checagem em processos críticos;
- Identificar anomalias com maior precisão e antecedência
